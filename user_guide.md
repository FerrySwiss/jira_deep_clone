# Deep Clone for Jira - User Guide

Welcome to **Deep Clone for Jira**. This guide will help you understand how to use the app to efficiently replicate and move issues within your Jira instance.

## 1. Getting Started
Once the app is installed, you don't need to configure anything. The "Deep Clone" action is integrated directly into your Jira issue view.

## 2. How to Clone an Issue
1. Open any Jira issue you wish to clone.
2. Click the **Actions (...)** button in the top-right corner of the issue.
3. Select **Deep Clone Issue** from the dropdown menu.
4. A popup window will appear with your cloning options.

## 3. Configuration Options
In the Deep Clone popup, you can customize your clone:

*   **Target Project**: Select the project where you want to create the new issue.
*   **Target Issue Type**: Select the issue type (e.g., Task, Bug, Story).
*   **Clone Sub-tasks**: If checked, all sub-tasks from the source issue will be recreated in the target project.
*   **Clone Comments**: Includes the full comment history in the new issue.
*   **Clone Attachments**: Copies files attached to the source issue.
*   **Clone Linked Issues**: Re-establishes links to other Jira issues.

## 4. Platform Limitations
To ensure system stability and high performance, the following limits apply:
*   **Attachment Size**: Only files up to **20MB** per file will be cloned. Any files exceeding this limit will be skipped during the process.
